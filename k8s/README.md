
# Docker Compose → Kubernetes Migration Guide

This guide explains how to convert the existing `docker-compose.yml` setup into a **clean Kubernetes deployment**.

We will:

1. Convert the Compose file to Kubernetes manifests
2. Clean up the generated YAML
3. Replace autogenerated labels with readable ones
4. Apply the manifests to Kubernetes

---

# 1. Install Kompose

Kompose converts Docker Compose files to Kubernetes manifests.

## macOS

```bash
brew install kompose
```

## Linux

```bash
curl -L https://github.com/kubernetes/kompose/releases/latest/download/kompose-linux-amd64 -o kompose
chmod +x kompose
sudo mv kompose /usr/local/bin
```

Verify installation:

```bash
kompose version
```

---

# 2. Convert Docker Compose to Kubernetes YAML

Run:

```bash
kompose convert -f docker-compose.yml
```

This will generate Kubernetes manifests.

Example output:

```
├── app-deployment.yaml
├── app-service.yaml
├── db-deployment.yaml
├── db-service.yaml
├── postgres-data-persistentvolumeclaim.yaml
```

These represent:

| File | Purpose |
|-----|------|
| `app-deployment.yaml` | Application pod definition |
| `app-service.yaml` | Service exposing the application |
| `db-deployment.yaml` | PostgreSQL pod |
| `db-service.yaml` | PostgreSQL internal service |
| `postgres-data-persistentvolumeclaim.yaml` | Persistent storage for Postgres |

---

# 3. Clean the Generated YAML

Kompose generates labels like:

```yaml
io.kompose.service: app
```

These are **not ideal for production**.

Replace them with clearer labels like:

```yaml
app: obelisk
```

---

# Example: Clean Deployment Labels

### Before (Kompose generated)

```yaml
metadata:
  labels:
    io.kompose.service: app
  name: app
```

### After (cleaned)

```yaml
metadata:
  name: obelisk
  labels:
    app: obelisk
```

---

# 4. Update Deployment Spec

Ensure the selector and pod template match the new labels.

```yaml
metadata:
  name: obelisk
  labels:
    app: obelisk

spec:
  replicas: 1

  selector:
    matchLabels:
      app: obelisk

  template:
    metadata:
      labels:
        app: obelisk

    spec:
      containers:
        - name: obelisk
          image: obelisk:latest
          ports:
            - containerPort: 8080
```

---

# 5. Clean the Service

### Before

```yaml
metadata:
  labels:
    io.kompose.service: app
```

### After

```yaml
metadata:
  name: obelisk
  labels:
    app: obelisk
```

Ensure the selector matches:

```yaml
spec:
  selector:
    app: obelisk
```

---

# 6. PostgreSQL Deployment

Use a separate label for the database:

```yaml
metadata:
  name: obelisk-db
  labels:
    app: obelisk-db
```

And ensure selectors match:

```yaml
selector:
  matchLabels:
    app: obelisk-db
```

---

# 7. Apply to Kubernetes

Apply manifests:

```bash
kubectl apply -f .
```

Verify resources:

```bash
kubectl get pods
kubectl get svc
kubectl get pvc
```

---

# 8. Test the Deployment

Check pod status:

```bash
kubectl get pods
```

View logs:

```bash
kubectl logs deployment/obelisk
```

Port forward if needed:

```bash
kubectl port-forward svc/obelisk 8080:8080
```

Then open:

```
http://localhost:8080
```

---

# 9. Expected Result

Pods:

```
kubectl get pods

obelisk-xxxxx
obelisk-db-xxxxx
```

Services:

```
kubectl get svc

obelisk
obelisk-db
```

# 10. Recommended Improvements

### Add resource limits

```yaml
resources:
  requests:
    cpu: "100m"
    memory: "128Mi"
  limits:
    cpu: "500m"
    memory: "512Mi"
```

### Add readiness / liveness probes

```yaml
livenessProbe:
  httpGet:
    path: /
    port: 8080
  initialDelaySeconds: 10
  periodSeconds: 10
```

### Use Secrets for database credentials

Do not store DB passwords directly in deployment manifests.

### Use ConfigMaps for environment variables

Separate configuration from code.

---

# Summary

Workflow:

```
docker-compose.yml
        │
        ▼
kompose convert
        │
        ▼
Generated Kubernetes YAML
        │
        ▼
Clean labels / structure
        │
        ▼
kubectl apply
```

You now have a **basic Kubernetes deployment derived from Docker Compose**.
